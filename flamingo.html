Description:

As a software engineer intern at Everproof, one of my tasks was to build a customer relationship management app for the new version of the Everproof platform. I created Flamingo as a seperate app within the pre-existing Django app. Through this, I could intergrate directly with the data models using an MVC architecture.

Technologies used:

  • Django (a web framework built on Python)
  • Bootstrap (grid system and styling)
  • Github (version control)

MVC architecture, data models and ORM:

One of the greatest strengths of this project was the clean intergration with the data models. I made use of Django's predefined class-based views, simplifying the development process for reoccuring features (adding, editing, deleting objects). For more complicated interactions with the data models, I wrote method-based views.

Working simultaneously with another developer:

I learnt about the importance of branches within version control when collaborating in a team. I found that communication and clear objectives allowed for an efficient development process.

UI/UX design & product management:

I worked closely with the customer success staff - I met with the team regularly, so that I could understand the common flows (eg. adding a user to an organisation). I designed the CRM system so that these operations could be performed in a logical manner.

Proudest feature:

One of my greatest challenges with this project was to cleanly implement the documents page. In this page, all documents associated with a user were to be listed in categories, according to the organisations these documents were viewable by. I had tried a solution using Javascript, but it got fairly messy.

Eventually, I completed the page using just Django and HTML. The code was much cleaner and self-contained. The documents page is viewable in the video below. It was a rewarding feeling to overcome this challenge.

Note: Due to secret keys within the initial code, Everproof has cloned my work into a new repo.
